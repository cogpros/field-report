2 · Code review style

How strict should the agent be when it reads back its own diffs before committing? Light = "looks fine, ship it." Paranoid = "second-pass adversarial review with security scan, runs /crucible on anything touching auth."

Recommendation

For a brand-new project with no users, standard is the right starting point. Light leaves bugs in. Paranoid slows you down on throwaway code. Upgrade to paranoid once you have customers or before the first production deploy.

Light — fast feedback, light QA
Standard — typecheck + tests pre-commit
Paranoid — full adversarial review
